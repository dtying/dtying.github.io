State

In Redux, everything is about state. State shape is the first thing you should think about.

It’s a good idea to think of its shape before writing any code. What’s the minimal representation of your app’s state as an object?

I had normalized the state shape to ensure data consistency. If you’re not sure what normalize is, you can check Dan’s video. To put it simply, it means the data is stored in a look-up table style, and referred to by its id.

Reducer

State object forms the structure of the root reducer.

As state and reducer lay the foundation of the web application logic, it’s where you should start thinking first.
One thing to note, it’s better to flatten the state object, rather than nest it. When you need to add new property in state object, try to compose it with multiple reducers, instead of adding property deep inside it. In this way, your reducer will be easy to test and read.
